# Product-Fit Analysis Validation Checklist

## Pre-Analysis Validation

- [ ] Validated idea document has been loaded and reviewed
- [ ] Core problem statement is clearly extracted
- [ ] Target audience is defined
- [ ] Value proposition is documented

## Product Type Assessment

- [ ] All 9 product types have been evaluated
- [ ] Fit scores are calculated using the 5 criteria
- [ ] Scores are justified with reasoning (not arbitrary)
- [ ] Top candidates (score >= 3.0) are identified

## Competitive Research

- [ ] Each top candidate has competitive research completed
- [ ] At least 3 competitors identified per product type
- [ ] All competitor data has source citations with URLs
- [ ] Market gaps are documented with evidence

## Gap Analysis

- [ ] Gap scores are calculated for each candidate
- [ ] Gap size is supported by market evidence
- [ ] Differentiation opportunities are specific and actionable
- [ ] Growth trends are backed by data

## Scoring Integrity

- [ ] Combined scores are calculated correctly (Fit + Gap / 2)
- [ ] Products are ranked by final score
- [ ] Tier assignments match the criteria (>=4.0 = Tier 1, etc.)
- [ ] Not recommended products have clear reasoning

## Resource Analysis

- [ ] Resource requirements are realistic
- [ ] Initial investment estimates are reasonable
- [ ] Time-to-revenue estimates are justified
- [ ] Synergies between products are identified

## Roadmap & Sequencing

- [ ] Launch sequence is logical
- [ ] Phase assignments consider dependencies
- [ ] Timeline is achievable
- [ ] Synergies inform the sequence

## Output Quality

- [ ] At least 2 Tier 1 or Tier 2 products recommended
- [ ] Recommendations are actionable
- [ ] Positioning is specific and differentiating
- [ ] JSON handoff structure is complete

## Source Citations

- [ ] All competitor data has source URLs
- [ ] All market claims have citations
- [ ] Sources are current (within last 12 months preferred)
- [ ] No unsubstantiated claims

## Final Checks

- [ ] Document is complete (no placeholder text)
- [ ] Spelling and formatting are correct
- [ ] Handoff to BMP section is complete
- [ ] Document saved to correct output path
