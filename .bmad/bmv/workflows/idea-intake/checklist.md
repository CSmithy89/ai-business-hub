# Idea Intake Validation Checklist

## Problem Definition

- [ ] Problem statement is specific (not generic)
- [ ] Target customer segment is identified
- [ ] Problem frequency assessed (daily/weekly/monthly)
- [ ] Problem severity assessed (annoying/painful/critical)
- [ ] Current alternatives documented (including "do nothing")
- [ ] Why now timing justification provided

## Solution Clarity

- [ ] Solution described in one clear sentence
- [ ] 3-5 key features identified (must-haves)
- [ ] Nice-to-haves separated from core features
- [ ] Differentiation articulated clearly
- [ ] Product category identified
- [ ] Technical feasibility considered

## Founder Fit

- [ ] Founder expertise assessed
- [ ] Personal connection to problem explored
- [ ] Resource availability documented
- [ ] Commitment level understood
- [ ] Team strengths and gaps identified

## Timing & Context

- [ ] "Why now" question answered
- [ ] Supporting trends identified
- [ ] Timing risks considered
- [ ] External factors documented

## Assumptions Captured

- [ ] Desirability assumptions listed
- [ ] Feasibility assumptions listed
- [ ] Viability assumptions listed
- [ ] Risk levels assigned to each
- [ ] Validation methods suggested

## Initial Signals

- [ ] Quick Google search completed
- [ ] Existing competitors noted
- [ ] Market existence signals captured
- [ ] Red flags documented
- [ ] Green flags documented

## Document Quality

- [ ] All template sections completed
- [ ] No placeholder text remaining
- [ ] Session ID generated
- [ ] Next steps clearly defined
- [ ] Executive summary captures key points

## Ready for Validation

- [ ] Idea is specific enough to validate
- [ ] Clear hypotheses to test
- [ ] Validation path recommended
- [ ] User confirmed to proceed
