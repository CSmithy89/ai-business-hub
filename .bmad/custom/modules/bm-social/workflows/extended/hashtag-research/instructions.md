# Hashtag Research Workflow

## Purpose
Research, analyze, and optimize hashtag strategies per platform using the Scout (Trend Scout) agent.

## Context Variables
- `{{topic}}` - Topic or keyword area to research
- `{{platform}}` - Target platform for hashtags
- `{{industry}}` - Industry vertical for relevance
- `{{existing_hashtags}}` - Current hashtags to evaluate

## Prerequisites
- Scout agent enabled
- Platform analytics access
- Hashtag library initialized

## Execution Steps

### Step 1: Enter Research Topic
Define hashtag research scope:

```
HASHTAG RESEARCH

Research Depth:
( ) Quick scan (top 20 hashtags)
(x) Standard (top 50 + analysis)
( ) Deep dive (100+ with trends)
```

## Hashtag Best Practices

| Platform | Count | Placement | Best Practices |
|----------|-------|-----------|----------------|
| Twitter | 1-2 | End of tweet | Relevant, not trending for sake |
| LinkedIn | 3-5 | End of post | Industry + branded |
| Instagram | 5-15 | First comment | Mix of sizes |
| TikTok | 3-5 | Caption | Trending + niche |
| YouTube | 3-5 | Description | Searchable terms |

## Hashtag Categories

| Category | Purpose | Example |
|----------|---------|---------|
| Branded | Owned, trackable | #HYVVE |
| Industry | Visibility in niche | #SaaS |
| Trending | Ride momentum | #AI |
| Community | Join conversations | #StartupTwitter |
| Campaign | Track campaigns | #HYVVELaunch |

## Error Handling

| Error | Resolution |
|-------|------------|
| Banned hashtag | Remove, suggest alternative |
| Shadowbanned tag | Flag, research replacement |
| API limit | Cache results, retry later |
| Trend ended | Mark as outdated |

## Events Emitted
- `social.hashtag.researched` - Research complete
- `social.hashtag.set_created` - New set saved
- `social.hashtag.applied` - Tags added to content
