# Brand Guidelines Compilation Workflow Instructions

<critical>This workflow compiles outputs from strategy, voice, and visual workflows - do not recreate content</critical>
<critical>Guidelines must be actionable and specific - vague guidance is useless</critical>

## Purpose

This workflow compiles all brand identity components into a comprehensive, unified brand guidelines document. It does not create new content but organizes and presents existing outputs in a cohesive, usable format.

<workflow>

<step n="1" goal="Gather All Brand Components">
<action>Load brand-strategy-document.md</action>
<action>Load brand-voice-guidelines.md</action>
<action>Load visual-identity-guide.md</action>
<action>Verify all components are complete</action>

<verification>
| Component | Status | Key Elements |
|-----------|--------|--------------|
| Strategy | ✓/✗ | Archetype, positioning, personas, messaging |
| Voice | ✓/✗ | Attributes, tone spectrum, writing guidelines |
| Visual | ✓/✗ | Logo, colors, typography, imagery |
</verification>
</step>

<step n="2" goal="Structure Guidelines Document">
<action>Organize document into logical sections</action>

<structure>
1. **Brand Foundation**
   - Brand story/about
   - Mission, vision, values
   - Brand archetype
   - Positioning statement

2. **Visual Identity**
   - Logo system
   - Color palette
   - Typography
   - Imagery & photography
   - Visual elements

3. **Verbal Identity**
   - Brand voice
   - Tone spectrum
   - Writing guidelines
   - Messaging framework

4. **Applications**
   - Digital applications
   - Print applications
   - Social media
   - Templates

5. **Resources**
   - Asset downloads
   - Contact information
   - Version history
</structure>
</step>

<step n="3" goal="Compile Strategy Section">
<action>Extract brand foundation from strategy document</action>
<action>Include executive-friendly overview</action>

<include>
- Brand story (brief narrative)
- Archetype summary (not full analysis)
- Positioning statement
- Target audience overview
- Key messages
</include>
</step>

<step n="4" goal="Compile Visual Section">
<action>Include all visual specifications from visual-identity-guide</action>
<action>Organize for easy reference</action>

<include>
- Logo usage rules with visual examples
- Color specifications with codes
- Typography hierarchy
- Imagery guidelines
- Do's and don'ts with visuals
</include>
</step>

<step n="5" goal="Compile Verbal Section">
<action>Include voice and writing guidelines from brand-voice-guidelines</action>
<action>Organize for practical use</action>

<include>
- Voice attributes with examples
- Tone spectrum
- Writing style guide
- Vocabulary standards
- Content type guidelines
</include>
</step>

<step n="6" goal="Add Application Guidelines">
<action>Create specific application guidance</action>

<applications>
## Digital Applications
- Website
- Email
- Social media
- Mobile app (if applicable)

## Print Applications
- Business cards
- Letterhead
- Brochures
- Signage

## Templates
- Presentation template guidelines
- Document template guidelines
- Email signature guidelines
</applications>
</step>

<step n="7" goal="Create Quick Reference">
<action>Build one-page summary card</action>

<quick_reference>
- Logo (primary version)
- Colors (HEX codes)
- Fonts (names)
- Voice (3-4 words)
- Do's (top 3)
- Don'ts (top 3)
</quick_reference>

<output>
Generate quick-reference-card.md
</output>
</step>

<step n="8" goal="Validate Completeness">
<action>Run validation checklist</action>
<action>Verify all sections present</action>
<action>Check for consistency</action>

<compile_document>
Generate brand-guidelines.md as the comprehensive master document
</compile_document>
</step>

</workflow>

## Anti-Hallucination Protocol

- Only include content from existing workflow outputs
- Do not invent new brand elements
- Maintain consistency with source documents
- Reference source for any specifications
