KB-03 Production Release Protocol

verified Verified
Last updated by Sarah Jenkins
• Updated 2 hours ago •
schedule 5 min read

This document outlines the standard operating procedure for deploying hotfixes and scheduled releases to the production environment. Adherence to this protocol is mandatory for all Release Engineers.

Prerequisites link

Before initiating any deployment, ensure the following conditions are met:

  • All unit tests must pass in the Staging Environment.
  • Approval from at least one Senior Engineer via @LeadDevs.
  • Database migration scripts must be reviewed and versioned.

Deployment Steps link

Execute the deployment command from the master branch. The system will automatically trigger the CI pipeline.

Terminal 
npm run deploy --production --tag=v2.4.0
info
Note: If the deployment fails, the system will automatically attempt a rollback. Please monitor the Grafana Dashboard.

Rollback Procedures

In case of critical failure, manually trigger the rollback via the CLI:

kb-cli rollback --force
