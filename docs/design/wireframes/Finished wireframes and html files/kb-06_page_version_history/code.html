Authentication & Security Standards

Scribe Suggestion

The original introduction was overly technical. I've suggested a more concise version that aligns with our developer experience guidelines.

This document outlines the standard authentication protocols for all internal and external public-facing APIs. Following these guidelines ensures consistent security practices across our microservices architecture.

1. Token-Based Authentication

All API requests must be authenticated using JSON Web Tokens (JWT). The token should be passed in the Authorization header using the Bearer schema.

Authorization: Bearer <token>

Token Expiration Policies

  • Access tokens are valid for 24 hours 1 hour to minimize security risks.
  • Refresh tokens are valid for 7 days and can be used to obtain new access tokens.
  • Tokens are automatically revoked upon password reset events.

2. API Rate Limiting

To prevent abuse, all endpoints are rate-limited by IP address and User ID. The standard limits are:

Tier Requests / Minute Burst Capacity
Free 60 100 150
Pro 1,000 2,000
Enterprise Unlimited Custom
