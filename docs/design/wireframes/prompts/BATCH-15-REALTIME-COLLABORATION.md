### Key Elements

1. **Cursor Line**
   - Width: 2px solid
   - Color: User's assigned color
   - Height: Full line height of text
   - Position: Between characters
   - Animation: 530ms blink cycle

2. **Cursor Flag/Label**
   - Position: Above cursor, offset right
   - Background: User's color (solid)
   - Text: White, 11px semibold
   - Content: User's first name (or full name if short)
   - Pointer: Small triangle pointing to cursor
   - Auto-hide: Fades after 3s inactivity

3. **Selection Highlight**
   - Background: User's color at 30% opacity
   - Covers selected text range
   - Multi-line: Follows text wrapping
   - Label: Small pill at selection end
   - No border (fill only)

4. **Typing Indicator**
   - Extends cursor label
   - Text: "typing..."
   - Animated dots: Three bouncing dots
   - Timeout: Hides 1s after last keystroke
   - Position: Within cursor flag

5. **Color Assignment**
   - Colors from predefined palette
   - Assigned on user join
   - Persistent for session
   - Recycled when user leaves
   - Maximum 8 distinct colors, then repeats

6. **Cursor Transitions**
   - Movement: Smooth 50ms transition
   - Appearance: Fade in 150ms
   - Disappearance: Fade out 300ms
   - Flag show/hide: Fade with 200ms

7. **Z-Index Layering**
   - Text content: Base layer
   - Selections: Above text
   - Cursor lines: Above selections
   - Cursor flags: Top layer
   - Overlapping: Later cursor on top

8. **Performance Considerations**
   - Throttle cursor updates: 50-100ms
   - Batch selection updates
   - Use CSS transforms for animations
   - Limit visible cursors (e.g., 10 max)

### Style Notes

- Cursor colors must have good contrast on white background
- Labels use shadow for readability on any background
- Reduced motion: Disable animations if prefers-reduced-motion
- Mobile: Tap shows cursor labels for all visible users
- Accessibility: Screen reader announces remote user activity

### Key Elements

1. **Compact Presence Bar**
   - Location: Editor header, right-aligned
   - Avatar stack: Max 3 visible + overflow
   - Count text: "X editing" or "X viewing"
   - Sync status indicator
   - Clickable to expand

2. **Avatar Stack**
   - Circle avatars with white border
   - Overlap creates cascade effect
   - Colored ring matches cursor color
   - "+N" pill for overflow
   - Tooltip on hover shows name

3. **Expanded Panel**
   - Dropdown below compact bar
   - Width: 320px
   - Sections: Editing, Viewing, Recent
   - Each user row: Color dot, avatar, name, status
   - "You" identified in list

4. **User Row Design**
   - Height: 40px
   - Left: Cursor color dot (8px)
   - Avatar: 28px circle
   - Name: 13px semibold
   - Status: 12px gray, right-aligned
   - Hover: Light background

5. **Sync Status States**
   - Saved âœ“: Green checkmark, all synced
   - Saving: Spinner, changes pending
   - Offline: Red dot, disconnected
   - Reconnecting: Amber spinner
   - Unsaved changes: Amber warning

6. **Floating Pill**
   - Fixed position when scrolled
   - Minimal: Just avatars + count
   - Click expands full panel
   - Appears after 100px scroll

7. **Real-time Updates**
   - User joins: Avatar slides in
   - User leaves: Avatar fades out
   - Status change: Smooth transition
   - Count updates immediately

8. **Recent Users**
   - Shows users who left in last 5 mins
   - Grayed styling
   - "Left X minutes ago" text
   - Useful for context

### Style Notes

- Avatar border creates visual separation
- Colored rings subtle but visible
- Dropdown has subtle shadow
- Sync status uses semantic colors
- Accessibility: Announce changes to screen readers

### Key Elements

1. **Conflict Notification Banner**
   - Position: Sticky top of editor
   - Background: Amber 100 (#FEF3C7)
   - Left border: 4px Amber 500
   - Icon: âš ï¸ warning
   - [Resolve] button opens modal
   - Cannot be dismissed until resolved

2. **Resolution Modal**
   - Width: 800px (or 90% viewport)
   - Shows conflict count: "1 of 2"
   - Navigation: [< Prev] [Next >]
   - Three panels: Yours, Server, Diff

3. **Side-by-Side View**
   - Your version: Left panel
   - Server version: Right panel
   - Attribution: Who made the change
   - Equal width columns
   - Scrollable independently

4. **Diff View**
   - Below side-by-side
   - Shows combined diff
   - + lines: Added (green background)
   - - lines: Removed (red background)
   - Attribution in diff markers

5. **Resolution Options**
   - Radio group selection
   - Keep server version (recommended default)
   - Keep your version
   - Merge manually (opens editor)

6. **Manual Merge Editor**
   - Shows when "Merge manually" selected
   - Editable text area
   - Pre-populated with best guess merge
   - User can edit freely
   - Syntax highlighting if code

7. **Inline Conflict Marker**
   - Alternative to modal for single conflicts
   - Git-style markers (<<<, ===, >>>)
   - Amber border and background
   - Quick action buttons
   - Visible in document flow

8. **Resolution Summary**
   - Shown after all conflicts resolved
   - Lists resolution choices
   - Confirmation of sync complete
   - [Done] closes and returns to editor

9. **Conflict Types Handled**
   - Text replacement conflicts
   - Deletion vs modification
   - Insertion at same position
   - Structural changes (rare)

10. **Prevention Strategies**
    - Yjs CRDT handles most merges
    - Conflicts rare with real-time sync
    - Mainly offline â†’ online transitions
    - Show "last edited by" before editing

### Style Notes

- Amber color scheme indicates caution, not error
- Git-style markers familiar to developers
- Clear attribution helps user decide
- Default to server version (more recent)
- Provide undo for resolution choices
- Diff colors: Green for additions, Red for deletions

## Implementation Notes

### Yjs Integration Points

1. **Awareness API** - Used for cursor positions and user presence
2. **Y.Doc** - Shared document state
3. **Hocuspocus** - WebSocket server for sync
4. **Persistence** - Document state persisted to PostgreSQL

### Performance Considerations

1. Throttle cursor updates to 50-100ms
2. Batch awareness updates
3. Use CSS transforms for animations
4. Limit visible remote cursors
5. Lazy load presence details

### Accessibility

1. Announce remote user joins/leaves
2. Provide keyboard navigation for conflict resolution
3. Support reduced motion preferences
4. Screen reader labels for all interactive elements

### Error Handling

1. Graceful offline handling
2. Automatic reconnection
3. Queue changes when offline
4. Show unsaved indicator
5. Conflict detection on reconnect
