### Key Updates

1. **KB Integration Panel (New)**
   - Position: Right column of task properties
   - Section: "Linked KB Pages" with page list
   - Each page shows title and verified badge
   - [+ Link Page] opens KB page search
   - Click page to navigate to KB

2. **KB References Section (New)**
   - Shows pages that reference this task
   - Detected via task ID mentions (#TASK-123)
   - Shows snippet with context
   - Click to navigate to reference
   - Real-time update when pages mention task

3. **AI Assist Toggle (New)**
   - Position: In task properties section
   - Radio toggle: Auto / Manual
   - Auto: AI agents can update this task
   - Manual: Human approval required
   - Affects confidence-based routing

4. **Scribe Suggestions Panel (New)**
   - Position: Below description
   - Collapsible section with teal accent
   - Suggestions:
     - Generate documentation for task
     - Find related KB content
     - Create implementation notes
   - RAG context indicator shows available context

5. **RAG Context Indicator**
   - Shows count of relevant KB pages
   - Indicates AI has context for assistance
   - [View] shows list of context pages
   - Used when agent works on task

- KB integration panel uses teal left border
- Scribe suggestions have teal accent (#20B2AA)
- RAG indicator uses info blue (#3B82F6)
- Verified badge: Green checkmark

### Key Updates

1. **Workflow Mode Selection (New)**
   - Two cards: BUILD Mode vs Quick Kanban
   - BUILD: Full BMAD 7-phase methodology
   - Quick Kanban: Simple board for ongoing work
   - Card selection with radio indicator
   - Description of use cases

2. **Template Grid (Enhanced)**
   - 4Ã—2 grid of project templates
   - Each template tailored to project type
   - Templates include: task structure, milestones, KB pages
   - Blank template for custom projects
   - [+ Custom] to create new template

3. **KB Starter Pages (New)**
   - Master checkbox to enable/disable
   - List of KB page templates per project template
   - Pre-selected recommended pages
   - Optional additional pages
   - Checkboxes for individual selection
   - [Select All] [Deselect All] actions

4. **Template-KB Mapping**
   - Each project template defines starter KB pages
   - Pages created linked to project
   - Pages use KB templates (from KB-12)
   - Pre-populated with project name, date

5. **BMAD Integration**
   - BUILD mode: Creates BMAD phase structure
   - Shows phase milestones in timeline
   - Links to BMAD documentation
   - OPERATE mode phases if selected later

- Template cards use 12px border-radius
- Selected template has coral border
- KB pages checkbox list indented with gray background

### Key Elements

1. **Assignment Mode Selector**
   - Three options: Human Only, Hybrid, AI Primary
   - Radio button group
   - Description for each mode
   - Default: Hybrid (recommended)
   - Affects capacity split visibility

2. **Primary Assignee Card**
   - Shows selected human team member
   - Name, role, available capacity
   - [Change] button opens team picker
   - Avatar with online indicator

3. **AI Assistant Card**
   - Shows assigned AI agent
   - Default: Navi (can be changed)
   - Lists assistance capabilities
   - [Change] opens agent picker

4. **Capacity Split Slider**
   - Dual-handle slider
   - Shows: Human % | AI %
   - Numeric values and hours
   - Based on task estimate
   - Minimum: 10% each side

5. **AI Assistance Scope**
   - Checklist of allowed actions
   - Pre-checked based on task type
   - Includes approval-required items
   - Customizable per task

6. **Approval Settings**
   - Confidence threshold (85% default)
   - Action when below threshold
   - Radio group selection
   - Integrates with approval queue

7. **Task Card Display**
   - Shows hybrid indicator (ğŸ¤)
   - Both assignees with percentages
   - Visual capacity bar
   - Compact representation

8. **Capacity Bar Design**
   - Human portion: Gray fill
   - AI portion: Teal fill
   - Percentages at each end
   - Smooth color transition

### Key Elements

1. **Session Header**
   - Session name and status
   - Participant count
   - Stories remaining
   - [End Session] [Settings] buttons

2. **Current Story Card**
   - Story ID and title
   - Full description
   - Acceptance criteria
   - Related KB pages
   - [View Story] link to detail

3. **Sage Suggestion Panel**
   - AI estimate with reasoning
   - Based on: historical data, velocity, complexity
   - Reference to similar stories
   - [Details] expands full analysis
   - Green accent (Sage color)

4. **Card Selection**
   - Fibonacci sequence: 0, 1, 2, 3, 5, 8, 13, ?
   - Card size: 60px Ã— 80px
   - Selected card: Coral border, lift effect
   - ? card for "need discussion"
   - Click to select, click again to deselect

5. **Team Vote Display (Pre-Reveal)**
   - Avatar + name for each participant
   - Cards face-down (âœ“ or ...)
   - Status: Ready/Thinking
   - Sage participates as team member
   - Progress: X of Y ready

6. **Reveal Button**
   - Enabled when all votes in
   - Flips all cards simultaneously
   - Animated reveal effect
   - Host can force reveal early

7. **Post-Reveal View**
   - Cards flip to show values
   - Majority/outliers highlighted
   - Statistics: Average, Range, Consensus
   - Variance warning if high spread
   - Actions: Re-vote, Discuss, Accept, Custom

8. **Session Progress**
   - Progress bar
   - List of completed stories with estimates
   - Current story highlighted
   - Remaining stories grayed

9. **Real-time Features**
   - Vote indicators update live
   - Yjs-powered synchronization
   - Presence cursors in discussion
   - Timer option for voting

### Style Notes

- Cards use cream background with subtle shadow
- Selected card has coral border and lift
- Sage votes included but distinguishable (robot icon)
- Reveal animation: 3D flip effect
- High variance uses amber warning
- Discussion mode opens threaded chat
